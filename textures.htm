Текстуры

Текстура — растровое изображение, накладываемое на поверхность полигональной модели для придания ей цвета, окраски или иллюзии рельефа.

Приблизительно использование текстур можно легко представить как рисунок на поверхности скульптурного изображения. Использование текстур позволяет воспроизвести малые объекты поверхности, создание которых полигонами оказалось бы чрезмерно ресурсоёмким. Например, шрамы на коже, складки на одежде, мелкие камни и прочие предметы на поверхности стен и почвы.

Качество текстурированной поверхности определяется текселями — количеством пикселей на минимальную единицу текстуры. Так как сама по себе текстура является изображением, разрешение текстуры и её формат играют большую роль, которая впоследствии сказывается на общем впечатлении от качества графики в 3D-приложении.

Метод проецирования текстур для использования в трёхмерной графике впервые был предложен Эдвином Катмуллом в 1974 году.

Существует технология parallax mapping для создания трёхмерного описания текстурированной поверхности с использованием карт смещения.

Текстура — изображение, воспроизводящее визуальные свойства каких-либо поверхностей или объектов. В отличие от рисунка, к текстуре не применяются нормы и требования композиции, поскольку текстура сама по себе художественным произведением не является, хотя и может иногда выступать доминантой в художественном произведении.

Текстуру, окрестности всех точек которой визуально подобны друг другу, называют равномерной текстурой (гомогенной текстурой).

В компьютерной графике текстурами часто называют растровые цифровые изображения, содержащие текстурные элементы.

Текстуру часто ошибочно называют фоном. Понятие фона относится к перспективному месту на изображении, заднему плану. Текстура же в этом смысле — это изображение, визуально отображающее совокупность свойств поверхности какого-либо объекта — реального или вымышленного.

Понятия «текстура» и «фактура» применительно к свойствам какой-либо поверхности используются синонимично. Иногда словом "фактура" называют совокупность тактильных свойств, а текстурой — совокупность свойств визуальных. Однако за цифровым графическим изображением таких свойств закрепилось слово «текстура».

CanvasTexture

Создает текстуру из элемента canvas.
Это фактически тоже самое, что и базовый класс Texture, за исключением того, что свойство needsUpdate сразу же устанавливается как true.

Конструктор

CanvasTexture( canvas, mapping, wrapS, wrapT, magFilter, minFilter, format, type, anisotropy )
canvas -- HTML элемент canvas, из которого загружается текстура.
mapping -- аргумент определяет способ применения изображения к объекту. Тип объекта THREE.UVMapping. Другие возможные варианты смотрите в константах режима картирования.
wrapS -- значением по умолчанию является THREE.ClampToEdgeWrapping. Другие возможные варианты смотрите в константах режима укладки текстур.
wrapT -- значением по умолчанию является THREE.ClampToEdgeWrapping. Другие возможные варианты смотрите в константах режима укладки текстур.
magFilter -- определяет способ выборки текстуры, когда тексель занимает больше одного пикселя. Значением по умолчанию является THREE.LinearFilter. Другие возможные варианты смотрите в константах фильтров увеличения.
minFilter -- определяет способ выборки текстуры, когда тексель занимает меньше одного пикселя. Значением по умолчанию является THREE.LinearMipMapLinearFilter. Другие возможные варианты смотрите в константах фильтров минимизации.
format -- формат используемый в текстуре. Другие возможные варианты смотрите в константах формата.
type -- значением по умолчанию является THREE.UnsignedByteType. Другие возможные варианты смотрите в константах типа.
anisotropy -- число выборок, взятых через пиксель по оси с наибольшей плотностью текселей. The number of samples taken along the axis through the pixel that has the highest density of texels.
По умолчанию, значение данного аргумента равно 1. Более высокое значение дает менее размытый результат, чем базовая mip-карта, за счет большего количества используемых выборок текстуры.
Поиск максимально возможного значения анизотропии для GPU делается с помощью renderer.getMaxAnisotropy; это значение обычно является степенью 2.

Общие свойства смотрите в описании базового класса Texture. 

.needsUpdate
Значением по умолчанию является true. Это необходимо для загрузки данных холста (canvas).

CompressedTexture

Создает текстуру, основываясь на данных в сжатом виде, например из DDS-файла.
Используется с CompressedTextureLoader.

Конструктор

CompressedTexture( mipmaps, width, height, format, type, mapping, wrapS, wrapT, magFilter, minFilter, anisotropy )
mipmaps - массив mip-карт должен содержать объекты с данными, шириной и высотой. mip-карты должны быть правильного формата и типа.
Примечание переводчика: mip-карта или mip-пирамида (mipmap) - множественное отображение (последовательность текстур одного и того же изображения с уменьшающимся разрешением по мере удаления отображаемого объекта от наблюдателя). Каждая из этих текстур называется mip-уровнем (англ. mip level) или уровнем детализации (англ. level of detail).

CubeTexture

Создает кубическую текстуру, состоящую из шести изображений.

Пример

var loader = new THREE.CubeTextureLoader();
loader.setPath( 'textures/cube/pisa/' );

var textureCube = loader.load( [
  'px.png', 'nx.png',
  'py.png', 'ny.png',
  'pz.png', 'nz.png'
] );

var material = new THREE.MeshBasicMaterial( { color: 0xffffff, envMap: textureCube } );

Конструктор

CubeTexture( images, mapping, wrapS, wrapT, magFilter, minFilter, format, type, anisotropy )
По функциональности и применению CubeTexture фактически равна Texture. Отличие состоит только в том, что аргумент images представляет собой массив из 6 изображений, а не одно изображение, и вариантами картирования является THREE.CubeReflectionMapping (по умолчанию) или THREE.CubeRefractionMapping.

DataTexture

Создает текстуру непосредственно из необработанных данных (raw data), ширины и высоты.

Конструктор

DataTexture( data, width, height, format, type, mapping, wrapS, wrapT, magFilter, minFilter, anisotropy )
The data argument must be an ArrayBuffer or a typed array view. Further parameters correspond to the properties inherited from [page:Texture, where both magFilter and minFilter default to THREE.NearestFilter. The properties flipY and generateMipmaps are intially set to false.

The interpretation of the data depends on type and format: If the type is THREE.UnsignedByteType, a Uint8Array will be useful for addressing the texel data. If the format is THREE.RGBAFormat, data needs four values for one texel; Red, Green, Blue and Alpha (typically the opacity). Similarly, THREE.RGBFormat specifies a format where only three values are used for each texel.
For the packed types, THREE.UnsignedShort4444Type, THREE.UnsignedShort5551Type or THREE.UnsignedShort565Type, all color components of one texel can be addressed as bitfields within an integer element of a Uint16Array.
In order to use the types THREE.FloatType and THREE.HalfFloatType, the WebGL implementation must support the respective extensions OES_texture_float and OES_texture_half_float. In order to use THREE.LinearFilter for component-wise, bilinear interpolation of the texels based on these types, the WebGL extensions OES_texture_float_linear or OES_texture_half_float_linear must also be present.

Texture

Создает текстуру применяемую к поверхности объекта или в качестве карты отражения или преломления.

Пример

// load a texture, set wrap mode to repeat
// загрузка текстуры, установка режима стыковки для повторения
var texture = new THREE.TextureLoader().load( "textures/water.jpg" );
texture.wrapS = THREE.RepeatWrapping;
texture.wrapT = THREE.RepeatWrapping;
texture.repeat.set( 4, 4 );

Конструктор

Texture( image, mapping, wrapS, wrapT, magFilter, minFilter, format, type, anisotropy )

Свойства


.anisotropy
Число выборок, взятых через пиксель по оси с наибольшей плотностью текселей. По умолчанию, значение данного аргумента равно 1. Более высокое значение дает менее размытый результат, чем базовая mip-карта, за счет большего количества используемых выборок текстуры.
Поиск максимально возможного значения анизотропии для GPU делается с помощью renderer.getMaxAnisotropy; это значение обычно является степенью 2.

.center
Свойство показывает, где находится центр вращения. Для поворота вокруг центральной точки установите данное значение как (0.5, 0.5). Значение по умолчанию равно (0.0, 0.0).

.encoding
Значением по умолчанию является THREE.LinearEncoding. Другие возможные варианты смотрите в описании констант кодирования текстуры.
Обратите внимание, что если данное значение изменяется на текстуре после использования материала, необходимо запустить Material.needsUpdate этого значения, чтобы выполнить обновление в шейдере.

.flipY
Значение по умолчанию равно true. Переворачивает координатную ось Y объекта для соответствия пространству текстурных координат WebGL.

.format
Значением по умолчанию является THREE.RGBAFormat, хотя для изображений в формате JPG TextureLoader автоматически установит это свойство как THREE.RGBFormat.
Другие возможные варианты смотрите в описании констант формата текстуры.

.generateMipmaps
Свойство определяет, будут ли создаваться для текстуры mip-карты (если это возможно). Значением по умолчанию является true.

.id
Уникальное число для данного экземпляра текстуры (только для чтения).

.image
Объект изображение, обычно создаваемый с помощью метода TextureLoader.load. Это может быть любой тип изображения (например, PNG, JPG, GIF, DDS) или видео (например, MP4, OGG/OGV), поддерживаемый three.js.
Чтобы использовать видео в качестве текстуры нужно иметь элемент HTML5, воспроизводящий видео, как ресурс для изображения текстуры и постоянно обновлять эту текстуру, до тех пор пока видео проигрывается - класс VideoTexture делает это автоматически.

.mapping
Свойство определяет способ применения изображения к объекту. По умолчанию типом объекта является THREE.UVMapping, то есть при укладке используются U и V координаты. Другие возможные варианты смотрите в константах режима картирования.br>
.magFilter
How the texture is sampled when a texel covers more than one pixel. The default is THREE.LinearFilter, which takes the four closest texels and bilinearly interpolates among them. The other option is THREE.NearestFilter, which uses the value of the closest texel.

.matrix : Matrix3
The uv-transform matrix for the texture. Updated by the renderer from the texture properties .offset : Vector2, .repeat : Vector2, and .rotation : numberwhen the texture's .matrixAutoUpdate : booleanproperty is true. When .matrixAutoUpdate : booleanproperty is false, this matrix may be set manually. Default is the indentity matrix.

.matrixAutoUpdate : boolean
Whether to update the texture's uv-transform .matrix : Matrix3based on the .offset : Vector2, .repeat : Vector2, and .rotation : numbersettings. True by default. Set this to false if you are specifying the uv-transform matrix directly.

.minFilter
How the texture is sampled when a texel covers less than one pixel. The default is THREE.LinearMipMapLinearFilter, which uses mipmapping and a trilinear filter. Other choices are THREE.NearestFilter, THREE.NearestMipMapNearestFilter, THREE.NearestMipMapLinearFilter, THREE.LinearFilter, and THREE.LinearMipMapNearestFilter. These vary whether the nearest texel or nearest four texels are retrieved on the nearest mipmap or nearest two mipmaps. Interpolation occurs among the samples retrieved.

.mipmaps
Массив mip-карт, задаваемый пользователем (дополнительное, необязательное свойство).

.name
Дополнительное имя объекта (ему необязательно быть уникальным). Значением по умолчанию является пустая строка.

.needsUpdate
Установите значение этого свойства как true для запуска обновления при следующем использовании текстуры. Особенно это важно для установки режима укладки текстуры.

.offset
Свойство показывает насколько одно повторение текстуры смещено от начала, в каждом направлении U и V. Обычный диапазон значений от 0.0 до 1.0.
Примечание: свойство offset является расчетным модификатором и влияет только на применение текстуры к первому набору координат UV модели. Если текстура используется как карта, требующая дополнительных наборов координат UV (например aoMap или lightMap большинства материалов ассортимента), то для достижения желаемого повторения эти координаты должны быть назначены вручную.

.onUpdate
Функция обратного вызова, вызываемая при обновлении текстуры (например когда свойство needsUpdate установлено как true и затем данная текстура используется).

.premultiplyAlpha
Значением по умолчанию равно false, что является нормой для изображений в формате PNG. Если значения RGB хранились предварительно перемноженными на значение альфа-канала (канал прозрачности), то установите значение true.

.repeat
Свойство показывает сколько раз текстура повторяется по поверхности, в каждом направлении U и V. Если значение этого свойства установлено больше 1 в любом направлении, то соответствующий параметр укладки текстуры (wrapS или wrapT) также должен быть установлен как THREE.RepeatWrapping или THREE.MirroredRepeatWrapping для достижения желаемого эффекта мозаичной укладки.
Примечание: свойство repeat является расчетным модификатором и влияет только на применение текстуры к первому набору координат UV модели. Если текстура используется как карта, требующая дополнительных наборов координат UV (например aoMap или lightMap большинства материалов ассортимента), то для достижения желаемого повторения эти координаты должны быть назначены вручную.

.rotation
Угол, на который текстура поворачивается вокруг центральной точки, в радианах. Положительным значениям соответствует поворот против часовой стрелки. Значение по умолчанию равно 0.

.type
Значение данного свойства должно соответствовать значению свойства format. Значением по умолчанию является THREE.UnsignedByteType, который будет использоваться для большинства форматов текстур.
Другие возможные варианты смотрите в описании констант типа текстуры.

.unpackAlignment
Свойство определяет требования к выравниванию в памяти начала каждого ряда пикселов. Значение по умолчанию равно 4. Допустимыми значениями являются 1 (выравнивание по байтам), 2 (ряды выравниваются по четным байтам), 4 (выравнивание по словам) и 8 (ряды начинаются на границах двойных слов). Подробнее смотрите в описании свойства glPixelStorei на www.khronos.org.

.uuid
UUID данного экземпляра объекта. Он присваивается автоматически и не должен быть изменен.
Вот статья Википедии про UUID на русском языке.

.version
Это свойство ведет подсчет (начиная с 0) сколько раз свойство needsUpdate было установлено как true.

.wrapS
Свойство определяет укладку текстуры по горизонтали и соответствует координате U в картировании по координатам UV. Значением по умолчанию является THREE.ClampToEdgeWrapping, при этом край прижимается к внешним текселям края. Другие возможные варианты - THREE.RepeatWrapping и THREE.MirroredRepeatWrapping. Подробности смотрите в описании констант режима укладки текстур.

.wrapT
Свойство определяет укладку текстуры по вертикали и соответствует координате V в картировании по координатам UV. Значение по умолчанию и другие возможные варианты такие же как у свойства wrapS.
Примечание: мозаичное размещение изображений в текстурах работает только в том случае, если размеры изображения являются степенью двойки (2, 4, 8, 16, 32, 64, 128, 256, 512, 1024, 2048, ...) в переводе на пиксели. Отдельные размеры не обязательно должны быть равными, но каждый из них должен быть степенью двойки. Это ограничение от WebGL, а не three.js.

VideoTexture

Creates a texture for use with a video texture.
This is almost the same as the base [page:Texture Texture class, except that it continuosly sets [page:Texture.needsUpdate needsUpdate to *true* (using requestAnimationFrame) so that the texture is updated as the video plays. Automatic creation of [page:Texture.mipmaps mipmaps is also disabled.

Пример

//assuming you have created a HTML video element with id="video"
var video = document.getElementById( 'video' );

var texture = new THREE.VideoTexture( video );
texture.minFilter = THREE.LinearFilter;
texture.magFilter = THREE.LinearFilter;
texture.format = THREE.RGBFormat;
Другие примеры
WebGL / materials / video

Конструктор

VideoTexture( video, mapping, wrapS, wrapT, magFilter, minFilter, format, type, anisotropy )
video -- видеоэлемент для использования в качестве текстуры.
mapping -- How the image is applied to the object. An object type of [page:Textures THREE.UVMapping. See [page:Textures mapping constants for other choices.
wrapS -- The default is [page:Textures THREE.ClampToEdgeWrapping. See [page:Textures wrap mode constants for other choices.
wrapT -- The default is [page:Textures THREE.ClampToEdgeWrapping. See [page:Textures wrap mode constants for other choices.
magFilter -- способ выбора текстуры, если тексель занимает больше одного пикселя. Значением по умолчанию является THREE.LinearFilter. Другие возможные варианты смотрите в константах фильтров увеличения.
minFilter -- способ выбора текстуры, если тексель занимает менее одного пикселя. Значением по умолчанию является THREE.LinearMipMapLinearFilter. Другие возможные варианты смотрите в константах фильтров уменьшения.
format -- формат используемый в текстуре. Возможные варианты смотрите в константах форматов текстур.
type -- значением по умолчанию является THREE.UnsignedByteType. Другие возможные варианты смотрите в константах типов текстур.
anisotropy -- The number of samples taken along the axis through the pixel that has the highest density of texels. По умолчанию это значение равно 1. A higher value gives a less blurry result than a basic mipmap, at the cost of more texture samples being used. Используйте метод renderer.getMaxAnisotropy( ), чтобы найти максимальное допустимое значение анизотропии для графического процессора (GPU); это значение обычно является степенью двойки.
